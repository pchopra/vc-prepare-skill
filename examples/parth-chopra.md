### They'll Probably Ask

- **"How is this different from Runway, Sora, or other AI video generation tools?"**
  Those tools generate discrete clips on demand: you submit a prompt, wait 90 seconds to 5 minutes, and download a 5-25 second video. Dazzle generates continuously as a live stream. An AI agent loop writes scripts, generates scenes (image/video/speech in parallel via LTX-2), and streams the output as a live channel. It is the difference between a photo printer and a TV station. Show him a running channel.

- **"How do you make money on this?"**
  Credit-based billing via Stripe, already implemented. Users buy credits, streaming consumes credits per GPU-minute. Three revenue surfaces: creators paying for 24/7 channels (replacing $50-125/month in fragmented tools), developers paying for MCP API access to give their agents visual output, and brands paying per-screen-per-month for always-fresh ambient content (replacing $2,000-10,000/quarter agency production cycles). Neuro-sama proves the creator tier alone can support $400K+/month per channel.

- **"What's the moat? Can't anyone wrap an AI video model in a streaming layer?"**
  The generation pipeline is the hard part. Each step involves plan generation, parallel script/scene/asset/speech production, speculative pre-generation of branching choices, real-time streaming, and an autonomous agent loop that maintains narrative coherence across hours of content. The MCP API with 17 tools, the agent lifecycle management, and the real-time event system (content_low, chat_activity, billing) create a platform other agents build on. This is not a wrapper; it is an operating system for AI-driven live video.

- **"Who are the first customers?"**
  Three wedges. First: AI developers and agent builders in the MCP ecosystem (50,000-200,000 globally). Dazzle is the only MCP server that gives an agent a live visual output channel. Second: lofi/ambient creators and 24/7 streamers who currently spend 8-12 hours per cycle rebuilding content loops. Third: Twitch streamers whose channels are dark 18+ hours/day. The developer wedge is the GTM priority because developers are both customers and distribution (they build agents that pull consumer adoption behind them).
